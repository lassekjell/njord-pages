5 The Method

Three layers of context

What determines whether AI delivers 50% or 99% accuracy isn't the prompt — it's the structure around it.

01
Permanent identitet
Permanent identity

Huvudprojektet "Njord" har en CLAUDE.md som alltid är aktiv. Den beskriver vem jag är, min befattning, mina beslutsvägar och hur Claude ska arbeta: alltid i planeringsläge först, alltid som arkitekt — inga antaganden utan mitt godkännande.

The main project "Njord" has a CLAUDE.md that's always active. It describes who I am, my role, my decision paths and how Claude should work: always in planning mode first, always as architect — no assumptions without my approval.

02
Projektkatalog
Project directory

Separata kataloger under Njord: branding (typsnitt, färger, templates), en per fartyg med teknisk detalj, en per projekt. Varje katalog har sin egen CLAUDE.md som pekar till rätt Markdown-filer: "för propelleranalys, läs denna fil", "motordata och sjöprov ligger här".

Separate directories under Njord: branding (fonts, colors, templates), one per vessel with technical detail, one per project. Each directory has its own CLAUDE.md pointing to the right Markdown files: "for propeller analysis, read this file", "engine data and sea trials are here".

03
Sessionsdisciplin
Session discipline

Jag ber aldrig Claude att söka information själv — det är tidsödande och blir lätt fel. Istället matar jag in verifierad dokumentation. Vi börjar alltid i planeringsfas: förstå problemet, kartlägg behoven, bygg strukturen. Först därefter producerar vi.

I never ask Claude to search for information itself — it's time-consuming and error-prone. Instead I feed verified documentation. We always start in planning phase: understand the problem, map the needs, build the structure. Only then do we produce.

Viktigaste principen
The key principle

Filer är output — Markdown är sanningen

Alla AI-verktyg kan producera en Excel-fil med mängder av data ur sitt minne efter en enda prompt. Men när du vill ändra en cell, en kolumn, en formel — då går det ofta helt fel. Du får börja om.

Min lösning: Excel-filen är bara en output. All data, alla antaganden, alla beräkningsgrunder ligger i Markdown-filer som Claude läser som källdata. Filen är ett resultat — inte källan.

Files are output — Markdown is the truth

Any AI tool can produce an Excel file with tons of data from memory after a single prompt. But when you want to change a cell, a column, a formula — it often goes completely wrong. You have to start over.

My solution: The Excel file is just an output. All data, all assumptions, all calculation bases live in Markdown files that Claude reads as source data. The file is a result — not the source.

5 Arbetsspråket

Markdown — enkelt & kraftfullt

5 The Language

Markdown — simple & powerful

Markdown is a simple text syntax that lets you focus on content instead of formatting. It's Claude's native language — and it should be yours too.

# Heading 1 ## Heading 2 Regular text with **bold** and *italic*. - Bullet list - With sub-items - Like this | Column A | Column B | |----------|----------| | Data 1 | Data 2 | Status: ✅ Done 🔴 Critical ⏳ In progress

Varför Markdown?

Versionshanterat

Textfiler kan enkelt spåras, jämföras och säkerhetskopieras. Du ser alltid vad som ändrats.

Konverterbart

Från .md till Word, PDF, HTML, PowerPoint. Ett format — många outputs.

AI-optimerat

Claude läser och skriver Markdown nativt. Det ger bäst resultat och snabbast arbetsflöde.

Why Markdown?

Version controlled

Text files can be easily tracked, compared and backed up. You always see what changed.

Convertible

From .md to Word, PDF, HTML, PowerPoint. One format — many outputs.

AI-optimized

Claude reads and writes Markdown natively. It gives the best results and fastest workflow.

7 Workflow

What does a session look like?

From idea to delivery — typical workflow when I work with Claude.

1. Öppna projektet

Peka Claude på rätt mapp. CLAUDE.md laddar automatiskt — kontexten är satt.

2. Beskriv uppgiften

Kort och konkret: "Skapa en riskmatris för hybridinstallationen" eller "Skriv mail till Scania om DW6-leverans".

3. Claude arbetar

Läser relevanta filer, ställer klargörande frågor, producerar materialet med rätt formatering.

4. Iterera & förfina

"Lägg till en kolumn för kostnad" eller "Gör tonen mer formell". Snabba justeringar.

5. Leverera

Färdigt dokument sparas direkt i projektet. Klart att använda, skicka eller dela.

1. Open the project

Point Claude to the right folder. CLAUDE.md loads automatically — context is set.

2. Describe the task

Short and concrete: "Create a risk matrix for the hybrid installation" or "Write an email to Scania about DW6 delivery".

3. Claude works

Reads relevant files, asks clarifying questions, produces the material with correct formatting.

4. Iterate & refine

"Add a column for cost" or "Make the tone more formal". Quick adjustments.

5. Deliver

Finished document saved directly in the project. Ready to use, send or share.
